# Compute the average difference between daily high and low SF temperatures and write it to /app/avg_temp.txt.

Bob was asked to compute the average daily high-minus-low temperature for San Francisco from two CSV files inside a Docker container and save the result to /app/avg_temp.txt. Its first move was a native read_file on /app/daily_temp_sf_high.csv, which failed because the path lay outside the workspace sandbox. Bob immediately recognized the files lived inside the container and switched to `docker exec ... cat` to read both CSVs. Inspecting them revealed the crux of the task: the high file used ISO dates (2025-04-19) in order, while the low file used heterogeneous formats (04/19/2025 06:00:00 and 04-23-2025 06:00:00) in shuffled row order. Rather than naively zipping rows, Bob wrote a one-shot Python script that parsed both date formats with a fallback loop, keyed both files by parsed date, matched on date, computed per-day differences, and averaged them. The script printed 11.428571428571429, which Bob wrote to the output file and then verified with a cat. It closed with attempt_completion reporting ~11.43F over 7 days.
