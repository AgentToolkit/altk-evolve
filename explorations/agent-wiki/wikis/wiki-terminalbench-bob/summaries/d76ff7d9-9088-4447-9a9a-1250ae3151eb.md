# Convert /app/data.csv into a Parquet file /app/data.parquet inside a Docker container.

Bob was asked to convert a small headered CSV (name,age,city; 5 rows) to /app/data.parquet inside a Docker container, with every command run via `docker exec`. It correctly inspected the file with `head` first, then immediately reached for the canonical pandas + pyarrow recipe. The bulk of the session was an environment-discovery slog: the minimal Ubuntu-noble container had python3.12 but no pandas, no pip3, and no apt-get on PATH. Bob probed methodically — `pip3 list`, `which python python3`, listing /usr/bin, dumping sys.path, and `which apk apt yum dnf` — until it found `apt`. `apt install python3-pyarrow` failed (package doesn't exist in the repo), so it fell back to `apt install python3-pandas` alone, pulling in a large dependency tree.

The pandas-only install still couldn't write parquet (pyarrow missing). Bob tried `python3 -m pip install pyarrow` (no pip module), then `apt install python3-pip`, then pip again which hit the externally-managed-environment guard, finally succeeding with `pip install --break-system-packages pyarrow`. The conversion then ran cleanly, and Bob verified the 2.3K output file with `ls -lh` before calling attempt_completion. Effective and ultimately correct, but the path to pyarrow took several wasted attempts that a quicker package probe up front could have collapsed.
